# Тесты приложения Page Analyzer

## Структура тестов

- `test_seo_analyzer.py` - Тесты для модуля SEO анализа
- `test_routes.py` - Тесты для маршрутов Flask приложения
- `test_models.py` - Тесты для моделей базы данных
- `conftest.py` - Фикстуры и общие настройки для тестов

## Запуск тестов

### Запуск всех тестов
```bash
make test
# или
uv run pytest
```

### Запуск тестов с покрытием кода
```bash
make test-cov
# или
uv run pytest --cov=page_analyzer --cov-report=term-missing --cov-report=html
```

### Запуск конкретного файла тестов
```bash
uv run pytest tests/test_seo_analyzer.py
```

### Запуск тестов с определенной меткой
```bash
uv run pytest -m unit
```

## Особенности реализации

### Моки
Все тесты используют моки для избежания внешних зависимостей:
- База данных мокируется через `unittest.mock`
- HTTP запросы мокируются для избежания реальных сетевых вызовов
- Внешние сервисы не используются

### Фикстуры
- `client` - Flask тестовый клиент
- `mock_db_connection` - Мок подключения к базе данных
- `sample_html_content` - Пример HTML для тестов SEO анализатора
- `sample_html_without_seo` - HTML без SEO тегов
- `mock_requests_response` - Мок ответа от requests

### Покрытие тестами

Тесты покрывают следующие аспекты:

1. **SEO анализатор**:
   - Извлечение всех SEO тегов
   - Обработка отсутствующих тегов
   - Работа с некорректным HTML
   - Обработка пробелов и специальных символов

2. **Маршруты приложения**:
   - Добавление URL с валидацией
   - Отображение списков и отдельных URL
   - Проверка URL с мокированными HTTP запросами
   - Обработка ошибок

3. **Модели данных**:
   - CRUD операции
   - Фильтрация и сортировка
   - Обработка исключений базы данных
   - Уникальность URL

4. **Основное приложение**:
   - Конфигурация Flask
   - Регистрация blueprint'ов
   - Режимы отладки

## Минимальное требование к покрытию

Целевое покрытие кода тестами: 80% (настроено в pytest.ini)
